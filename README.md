Financial Data Warehouse & Reporting Project

1. Project Overview: 
This project builds a financial data warehouse using banking transaction data. The goal is to transform raw financial transactions into structured fact and dimension tables for reporting and analytics.

2. Tools Used
- Python (Pandas)
- PostgreSQL
- SQL
- Power BI
- GitHub

3. Data Pipeline: 
Raw CSV → Python Cleaning → PostgreSQL Raw Table → Dimension Tables → Fact Table → SQL Analytics → Power BI Dashboard

4. Data Model
- dim_customer
- dim_account
- dim_transaction_type
- dim_date
- fact_transactions

5. Key Features
- ETL pipeline
- Star schema design
- Data quality checks
- Fraud trend analysis
- Financial reporting dashboard

6. Business Questions Answered
- What is total transaction volume?
- What transaction types generate the highest amount?
- What is the monthly transaction trend?
- What is the fraud rate?
- Which customers have the highest transaction amount?
